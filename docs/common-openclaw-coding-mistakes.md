# Common OpenClaw Coding Mistakes

## 1. Asking for code before a blueprint
This is how convincing sludge gets created.

## 2. Letting `AGENTS.md` turn into a junk drawer
If your operating rules are bloated, your context gets bloated too.

## 3. Using memoization like seasoning
`memo`, `useMemo`, and `useCallback` are tools, not confetti.

## 4. Pushing too much into client components
That is an easy way to bloat the bundle and widen the hydration surface.

## 5. Shipping APIs with no latency or query budget
Then acting shocked when localhost lied to you.

## 6. Letting durable memory become a landfill
Curate it. Do not dump everything into it.

## 7. Declaring something optimized without measurement
That is not engineering. That is optimism with props.

## 8. Keeping one giant chat thread for everything
Use focused tasks and focused specs.
